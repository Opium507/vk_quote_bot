# VK Quote Bot

Бот для ВКонтакте, который делает карточки-цитаты из пересланных сообщений.

Поддерживает два режима:

- **Одиночная цитата** — рендер одного пересланного сообщения в выбранном стиле.
- **Тред** — рендер до 5 пересланных сообщений в едином messenger-style оформлении.

## Возможности

- Генерация цитат из пересланных сообщений.
- Отдельный режим `!цитата` для одной цитаты.
- Отдельный режим `!цитата все` для треда из нескольких сообщений.
- Переключение стиля одиночных цитат.
- Независимый стиль для тредов.
- Работа в личных сообщениях с ботом.

## Команды

- `!помощь` — показать инструкцию.
- `!стиль` — открыть меню выбора стиля одиночных цитат.
- `!цитата` — создать карточку из одного пересланного сообщения.
- `!цитата все` — создать тред из пересланных сообщений, до 5 штук.
- `!ц список` — команда суперадмина для вывода списка добавленных айдишников кто может пользоваться ботом.
- `!добавить/удалить` — соответственно добавляет/удаляет людей в список кто моджет пользоваться ботом

## Как пользоваться

### Одиночная цитата

1. Откройте диалог с ботом.
2. Перешлите **одно** сообщение.
3. В том же сообщении сообщении отправьте команду `!цитата`.
4. Бот сгенерирует карточку в текущем выбранном стиле.

### Тред

1. Откройте диалог с ботом.
2. Перешлите до **5 сообщений**.
3. Отправьте команду `!цитата все` вместе с пересланными сообщениями.
4. Бот соберет тред в едином стиле.

## Логика стилей

В проекте есть два разных визуальных режима:

### 1. Single-режим

Используется для команды `!цитата`.

Для него можно выбирать стиль: например, `классический`, `пиксель`, `хакер`.
Параметры таких стилей лежат в `app\render\themes\styles.json`.

### 2. Thread-режим

Используется для команды `!цитата все`.

Этот режим не зависит от выбранного single-стиля. Для него используется один фиксированный стиль треда: черный фон, серые bubble-блоки, синий цвет имени автора, светлый текст и курсивный заголовок.

## Как добавить свой стиль

Одиночные стили удобно хранить в JSON-файле. Один объект = один стиль.

Пример структуры:

```json
{
  "classic": {
    "key": "classic",
    "display_name": "Классический",
    "title": "Цитаты великих людей",
    "background_color": [0, 0, 0],
    "canvas_color": [0, 0, 0],
    "text_color": [245, 245, 245],
    "muted_text_color": [200, 200, 200],
    "accent_color": [245, 245, 245],
    "bubble_color": [18, 18, 18],
    "bubble_text_color": [245, 245, 245],
    "name_color": [245, 245, 245],
    "separator_color": [40, 40, 40],
    "title_font": "app/assets/fonts/Inter-Regular.otf",
    "body_font": "app/assets/fonts/Inter-Regular.otf",
    "author_font": "app/assets/fonts/Inter-Regular.otf",
    "quote_prefix": "«",
    "quote_suffix": "»",
    "author_prefix": "© ",
    "title_font_size": 56,
    "body_font_size": 64,
    "author_font_size": 42,
    "max_text_width": 940,
    "line_spacing": 16
  }
}
```

## Описание полей

| Поле | Что делает | Как влияет на результат |
|---|---|---|
| `key` | Внутренний идентификатор стиля. | Должен быть уникальным. Обычно совпадает с ключом объекта, например `classic`. |
| `display_name` | Отображаемое название стиля. | Показывается пользователю в меню выбора стиля. |
| `title` | Заголовок карточки. | Это верхняя надпись на изображении, например `Цитаты великих людей`. |
| `background_color` | Основной цвет фона. | Используется как базовый фон стиля. Формат — RGB-массив `[R, G, B]`. |
| `canvas_color` | Цвет холста. | Обычно совпадает с `background_color`, но можно использовать отдельно, если фон и холст должны отличаться. |
| `text_color` | Основной цвет текста цитаты. | Главный цвет для самой цитаты. |
| `muted_text_color` | Вторичный приглушенный текст. | Подходит для менее акцентных подписей, второстепенного текста или дополнительной информации. |
| `accent_color` | Акцентный цвет. | Используется для выделений, декоративных деталей или важных текстовых элементов. |
| `bubble_color` | Цвет плашки или текстового блока. | Чем светлее значение, тем заметнее bubble-блок на фоне. |
| `bubble_text_color` | Цвет текста внутри bubble-блока. | Должен хорошо контрастировать с `bubble_color`. |
| `name_color` | Цвет имени автора. | Позволяет отдельно выделять автора цитаты. |
| `separator_color` | Цвет разделителей и тонких линий. | Подходит для декоративных линий, границ и визуального разделения элементов. |
| `title_font` | Путь к шрифту заголовка. | Используется только для верхнего заголовка карточки. |
| `body_font` | Путь к шрифту основного текста. | Используется для текста цитаты. |
| `author_font` | Путь к шрифту автора. | Используется для подписи автора. |
| `quote_prefix` | Префикс перед текстом цитаты. | Например `«`. Можно заменить на пустую строку, если кавычки не нужны. |
| `quote_suffix` | Суффикс после текста цитаты. | Например `»`. Можно убрать для более минималистичного вида. |
| `author_prefix` | Префикс перед именем автора. | Например `© `, `— `, `@ ` или пустая строка. |
| `title_font_size` | Размер шрифта заголовка. | Чем больше значение, тем крупнее верхний заголовок. |
| `body_font_size` | Размер текста цитаты. | Главный параметр читаемости и визуального веса карточки. |
| `author_font_size` | Размер подписи автора. | Помогает сделать автора заметнее или, наоборот, второстепенным. |
| `max_text_width` | Максимальная ширина текстового блока. | Чем меньше значение, тем раньше текст будет переноситься на новую строку. |
| `line_spacing` | Межстрочный интервал. | Чем больше значение, тем “воздушнее” выглядит длинная цитата. |

## Как правильно заполнять поля

### Цвета

Все цветовые поля задаются в формате RGB:

```json
"text_color": 
```

Где:

- `245` — красный канал;
- `245` — зеленый канал;
- `245` — синий канал.

Диапазон каждого значения: от `0` до `255`.

Примеры:

- `[0, 0, 0]` — черный;
- `[255, 255, 255]` — белый;
- `[120, 170, 255]` — мягкий голубой;
- `[40, 40, 40]` — темно-серый.

### Пути к шрифтам

Поля `title_font`, `body_font`, `author_font` должны содержать корректный путь к `.otf` или `.ttf` файлу:

```json
"title_font": "app/assets/fonts/Inter-Regular.otf"
```

Если путь указан неверно, рендер может завершиться ошибкой или стиль загрузится некорректно.

### Текстовые префиксы и суффиксы

Префиксы и суффиксы позволяют быстро менять подачу карточки без переписывания логики рендера.

Примеры:

```json
"quote_prefix": "«",
"quote_suffix": "»",
"author_prefix": "© "
```

Можно сделать и так:

```json
"quote_prefix": "",
"quote_suffix": "",
"author_prefix": "— "
```

## Как получить нужный визуальный результат

### Если нужен строгий минималистичный стиль

Используйте:

- темный фон;
- светлый текст;
- одинаковые или близкие цвета для `text_color`, `name_color`, `accent_color`;
- один шрифт без лишнего декоративного контраста;
- пустые или нейтральные префиксы.

Пример идеи:

- `background_color`: `[0, 0, 0]`
- `bubble_color`: `[18, 18, 18]`
- `text_color`: `[245, 245, 245]`

### Если нужен стиль с акцентом на автора

Сделайте `name_color` заметно отличающимся от основного текста.

Пример:

```json
"text_color": ,
"name_color": 
```

Так имя автора будет сразу бросаться в глаза.

### Если нужен “воздушный” стиль

Увеличьте:

- `max_text_width`;
- `line_spacing`.

Пример:

```json
"max_text_width": 980,
"line_spacing": 22
```

Так текст будет смотреться свободнее и легче читаться на длинных цитатах.

### Если нужен плотный и собранный стиль

Уменьшите:

- `max_text_width`, если нужен более узкий столбец текста;
- `line_spacing`, если хочется компактности;
- размеры шрифтов, если текст часто длинный.

Пример:

```json
"body_font_size": 56,
"max_text_width": 860,
"line_spacing": 12
```

### Если нужен более “плакатный” стиль

Увеличьте:
- `title_font_size`;
- `body_font_size`;
- контраст между фоном и текстом.

Пример:
```json
"title_font_size": 64,
"body_font_size": 72
```

Так карточка будет выглядеть агрессивнее и визуально мощнее.

## Практические советы по настройке

- Начинайте с копии существующего стиля, например `classic`.
- Меняйте по 1–2 параметра за раз, чтобы понимать, что именно повлияло на результат.
- Сначала настройте фон и текст, затем шрифты, потом размеры и интервалы.
- Если карточка выглядит “грязно”, обычно проблема в слабом контрасте, слишком большом количестве акцентных цветов или слишком тесном интерлиньяже.
- Если текст выглядит слишком тяжелым, уменьшите `body_font_size` или увеличьте `line_spacing`.
- Если автор теряется, увеличьте `author_font_size` или измените `name_color`.
- Если цитата получается слишком широкой, уменьшите `max_text_width`.

## Как подключить новый стиль

1. Добавьте новый объект в JSON-файл со стилями.
2. Укажите уникальный `key`.
3. Проверьте, что все пути к шрифтам существуют.
4. Убедитесь, что стиль подхватывается `StyleService`.
5. Перезапустите бота.
6. Выберите стиль через меню `!стиль`.## Где обычно используется JSON стилей

Обычно схема такая:

- `StyleService` читает JSON со стилями.
- По `user_id` хранится выбранный пользователем стиль.
- При вызове `!цитата` бот берет текущий стиль пользователя.
- При вызове `!цитата все` бот использует встроенный thread-style и игнорирует выбор single-стиля.

## Поддержка emoji

Бот рендерит emoji локально, через PNG‑ассеты Twemoji.  
Это позволяет сохранить внешний вид emoji на карточках и не зависеть от сети.

### Где лежат emoji

Ожидаемая структура:

```text
app/
  assets/
    emoji/
      1f600.png
      1f601.png
      1f602.png
      ...
      1f468-200d-1f4bb.png
```

- Папка: `app/assets/emoji/`.
- Формат имени файла: Unicode-кодпоинты в hex, разделённые дефисами, плюс `.png`.  
  Примеры:
  - `😎` → `1f60e.png`
  - `💀` → `1f480.png`
  - `🏴‍☠️` → `1f3f4-200d-2620-fe0f.png`

### Как подготовить emoji‑пак

1. Скачать Twemoji PNG (каталог `assets/72x72`) из репозитория Twemoji.

```text
https://github.com/twitter/twemoji/tree/master/assets/72x72
```

2. Скопировать все `.png` в `app/assets/emoji/`.

3. Следить, чтобы формат имён файлов сохранялся (`<codepoints>.png`).

### Как это работает в коде

- Функции рендера используют общий модуль `app/render/text_utils.py`.
- `text_utils.wrap_text(...)`:
  - учитывает ручные переносы (`\n`, `<br>`, `<br/>`, `<br />`);
  - делает автоматический перенос по заданной ширине.
- `text_utils.draw_multiline_text_with_local_emoji(...)`:
  - разбирает строку на текст и emoji;
  - для каждого emoji строит имя файла по Unicode;
  - ищет соответствующий PNG в `app/assets/emoji/`;
  - подгоняет размер emoji под высоту строки и вставляет его в изображение.

Если нужного emoji‑файла нет, символ рисуется как обычный текст — бот не падает и не ломает верстку.

## Пример сценария работы

### Выбор стиля

1. Пользователь отправляет `!стиль`.
2. Бот показывает кнопки со стилями.
3. Пользователь выбирает стиль.
4. Бот сохраняет выбор для этого пользователя.

### Создание одиночной цитаты

1. Пользователь пересылает сообщение.
2. Отправляет `!цитата`.
3. Бот строит `SingleRenderRequest`.
4. Рендерер генерирует изображение.
5. Бот загружает изображение в сообщения VK и отправляет его обратно.

### Создание треда

1. Пользователь пересылает несколько сообщений.
2. Отправляет `!цитата все`.
3. Бот строит `ThreadRenderRequest`.
4. Рендерер объединяет сообщения в один холст.
5. Бот отправляет готовую картинку.

## Обработка ошибок

Если рендер или отправка не удались, боту лучше отвечать понятным сообщением, например:

```python
def build_render_error_text() -> str:
    return "Не получилось собрать карточку. Попробуй еще раз."
```

Для сетевых ошибок при загрузке изображения в VK рекомендуется:

- оборачивать отправку фото в `try/except`;
- ловить `requests.exceptions.RequestException`;
- использовать 2–3 повторные попытки отправки;
- не давать одному неудачному запросу уронить весь процесс бота.

## Переменные окружения

Для локального запуска и деплоя бот читает настройки из файла `.env`.  
В репозитории есть шаблон `.env.example` — его нужно скопировать и заполнить:

```bash
cp .env.example .env
```

Шаблон:

```dotenv
VK_GROUP_TOKEN=your_group_token_here
VK_GROUP_ID=123456789
DEFAULT_STYLE=classic
MAX_FORWARD_COUNT=5
SUPERADMIN_ID=56789
```

Описание полей:

- `VK_GROUP_TOKEN` — токен доступа сообщества ВКонтакте.  
  Получается в настройках группы в разделе “Работа с API” / “Ключи доступа”. Токен должен иметь права на работу с сообщениями (бот для сообщества).

- `VK_GROUP_ID` — числовой ID сообщества, в котором работает бот.  
  Можно взять из настроек группы или из адресной строки (формат `https://vk.com/public<id>` или `club<id>` → используйте только число, без `public`/`club`).

- `DEFAULT_STYLE` — ключ стиля по умолчанию для одиночных цитат.  
  Должен совпадать с `key` в `app/render/themes/styles.json`, например `classic`, `pixel`, `hacker` и т.п. Если пользователь еще не выбирал стиль, будет использоваться именно он.

- `MAX_FORWARD_COUNT` — максимальное количество пересланных сообщений для режима `!цитата все`.  
  По умолчанию `5`. Если указать меньше, бот будет обрезать тред до этого количества пересланных сообщений.

- `SUPERADMIN_ID` — ваш VK ID

После заполнения `.env` достаточно перезапустить бота.

## Запуск через Docker (рекомендуемый способ)

### Требования

- Установленные Docker и Docker Compose (или Docker Desktop с включённым WSL2 на Windows).

### Подготовка

1. Склонируйте репозиторий и перейдите в папку проекта:

   ```bash
   git clone https://github.com/yourusername/vk_quote_bot.git
   cd vk_quote_bot
   ```

2. Скопируйте файл с переменными окружения и заполните его:

   ```bash
   cp .env.example .env
   nano .env   # или любой редактор
   ```

  Обязательно укажите VK_GROUP_TOKEN, VK_GROUP_ID, SUPERADMIN_ID (ваш VK ID).

3. Подготовьте папку с эмодзи (только если вы не хотите, чтобы образ был тяжёлым).
   Скачайте Twemoji PNG (папка 72x72) с GitHub Twemoji и положите все .png файлы в app/assets/emoji/.
   Если пропустить этот шаг, эмодзи будут отображаться как обычные символы (без картинок).

4. Убедитесь, что в папке app/assets/fonts лежат шрифты (они уже включены в репозиторий, если вы их добавили; иначе скачайте согласно инструкции выше).

### Запуск

```bash
docker-compose up -d --build
```

После сборки бот запустится в фоне. Проверить логи:

```bash
docker logs -f vk_quote_bot
```

Остановить бота

```bash
docker-compose down
```
